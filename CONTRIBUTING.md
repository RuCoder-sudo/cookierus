## Как сообщить о баге

**Перед созданием Issue:**

1. Убедитесь, что используете **последнюю версию** плагина — возможно, баг уже исправлен.
2. Поищите среди [открытых и закрытых Issues](https://github.com/RuCoder-sudo/cookierus/issues?q=is%3Aissue) — возможно, проблема уже обсуждается.
3. Если нашли уязвимость безопасности — **не создавайте публичный Issue**, см. [Сообщение об уязвимостях](#сообщение-об-уязвимостях).

**При создании Issue приложите:**

- Версию плагина CookieRus
- Версию WordPress и PHP
- Список активных плагинов и активную тему
- Подробное описание проблемы и шаги для воспроизведения
- Ожидаемое и фактическое поведение
- Скриншоты или видео (при необходимости)
- Логи из `wp-content/debug.log` или вкладки «Логи» плагина (если есть)

## Как предложить новую функцию

Создайте Issue с меткой **`enhancement`** и опишите:

- Какую задачу решает функция (что неудобно сейчас?)
- Как вы видите интерфейс/настройку
- Альтернативы, которые рассматривали
- Готовы ли вы взяться за реализацию

Большие изменения лучше сначала обсудить в Issue — так вы не потратите время на pull request, который может не подойти по архитектуре.

## Как прислать pull request

1. **Сделайте форк** репозитория и склонируйте его локально.
2. **Создайте ветку** от `main` с понятным именем:
   ```bash
   git checkout -b feature/cookie-icon-banner
   git checkout -b fix/banner-flicker-on-navigation
   ```
3. **Внесите изменения** — следуйте [стилю кода](#стиль-кода).
4. **Протестируйте** на чистой установке WordPress (см. [Локальная разработка](#локальная-разработка)).
5. **Обновите документацию** — `readme.txt`, `README.md`, `CHANGELOG.md`, если нужно.
6. **Сделайте коммит** с понятным сообщением:
   ```
   feat(banner): добавлена иконка cookie с настройкой размера
   fix(banner): устранено мерцание при переходах между страницами
   docs(readme): обновлён changelog для 1.0.5
   ```
7. **Отправьте pull request** в ветку `main` с описанием:
   - Что меняется и зачем
   - Связанный Issue (если есть): `Closes #42`
   - Скриншоты UI-изменений
   - Чек-лист, что вы проверили

### Чек-лист перед отправкой PR

- [ ] Код работает на минимальной версии (WordPress 5.0, PHP 7.4)
- [ ] Все строки, видимые пользователю, на русском языке
- [ ] Пользовательский ввод санитизирован (`sanitize_text_field`, `esc_html`, `wp_kses_post` и т. д.)
- [ ] AJAX-эндпоинты проверяют nonce (`check_ajax_referer`) и права (`current_user_can`)
- [ ] SQL-запросы используют `$wpdb->prepare()`
- [ ] Версия плагина обновлена в `CookieRus.php`, `readme.txt` и `CHANGELOG.md`
- [ ] Запись добавлена в `CHANGELOG.md` и `readme.txt`

## Стиль кода

### PHP

- Соблюдайте [WordPress Coding Standards](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/php/).
- Отступ — 4 пробела (не табы) для PHP.
- Префикс всех функций, классов и опций — `cookierus_` / `CookieRus_`.
- Все public-методы документируйте PHPDoc-комментариями.
- Никаких `extract()`, `eval()`, прямых SQL без `$wpdb->prepare()`.

### JavaScript

- Отступ — 4 пробела.
- Используйте `var`/`let` корректно, избегайте утечек в глобальную область.
- Не подключайте сторонние библиотеки — плагин должен оставаться лёгким.

### CSS

- Отступ — 4 пробела.
- Все классы префиксуйте `cookierus-`.
- Избегайте `!important`, если это не требуется для перебивки темы.

## Локальная разработка

1. Установите локальный WordPress (Local by Flywheel, XAMPP, Docker и т. п.).
2. Склонируйте репозиторий в `wp-content/plugins/`:
   ```bash
   cd wp-content/plugins
   git clone https://github.com/RuCoder-sudo/cookierus.git
   ```
3. Активируйте плагин в админке WordPress.
4. Включите режим отладки в `wp-config.php`:
   ```php
   define( 'WP_DEBUG', true );
   define( 'WP_DEBUG_LOG', true );
   define( 'WP_DEBUG_DISPLAY', false );
   define( 'SCRIPT_DEBUG', true );
   ```
5. Логи смотрите в `wp-content/debug.log`.
